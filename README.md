# 投资仪表板

> 本仪表板整合港股、北交所打新数据与可转债机器学习实盘数据，生成单文件 HTML 网页便于本地浏览或云端部署。

---

## 📁 文件结构

```
投资仪表板/
├── data.xlsx              # 整合后的数据源（用户可手动修改）
├── index.html             # 生成的仪表板网页（单文件，含所有数据）
├── generate_dashboard.py  # 数据整合与网页生成脚本
└── README.md              # 本文档
```

---

## 🔄 更新流程

### 方式 A：修改原始数据源后重新整合
适用于更新了以下源文件的情况：
- 家庭资产记录（投资日历）
- 北交所打新数据
- 港股新股数据
- ML 仪表盘 HTML

```bash
python generate_dashboard.py
```

> ⚠️ 此模式会**覆盖** `data.xlsx`。

### 方式 B：直接修改 data.xlsx 后快速生成网页
适用于只在 `data.xlsx` 中手动修改了数据的情况。

```bash
python generate_dashboard.py --from-excel
```

> ✅ 此模式**不会覆盖** `data.xlsx`。

#### 4. 可转债 ML 板块

  3. 模型表现追踪（OOS R² 综合）

- **修复方案**：使用否定前瞻 `((?:(?!</div>).)*?)` 限制匹配范围，标题长度恢复为正常的 25/35/19 字符

| 📅 打新日历 | 港股、北交所、其他事件的时间线 |

## ⚠️ 已知限制

1. **图表仅在完整模式下生成**：`--from-excel` 模式下图表需从原始 ML HTML 文件读取，如果原始文件路径变动，图表会缺失。
2. **港股数据只显示最近 50 只**：完整数据有 88 条，表格中只渲染最近 50 条以保持页面流畅。
3. **北交所日期**：data.xlsx 中若输入了带时间的日期（如 `2026-04-08 00:00:00`），脚本会自动截断为 `2026-04-08`。
